# Проверка контракта API перед frontend реализацией

## Когда применять
Когда нужно добавить на frontend фильтры, сортировку, поиск или другие параметры, которые планируется передавать в API запрос.

## Проблема
Frontend может отправлять параметры (`sort`, `filters`, `q`), которые backend **не поддерживает**. Backend просто игнорирует их и возвращает те же данные. На frontend — анимация обновления, но по факту ничего не меняется. Это вызывает реальный баг, который сложно заметить.

## Шаги

1. **Прочитать backend route handler** — найти endpoint (например `/api/places`), посмотреть какие query-параметры он реально принимает.

2. **Разделить параметры на два типа:**
   - **Server-side** — параметры, которые backend обрабатывает (например `category`, `lat`, `lng`). Передавать в API запрос, при изменении — делать новый `fetchAndRender()`.
   - **Client-side** — параметры, которые backend НЕ принимает (например `sort`, `filters`, `q`). Реализовать обработку на frontend.

3. **Реализовать client-side обработку:**
   - Хранить raw данные отдельно (`state.allPlaces`) и filtered (`state.places`)
   - Создать `applyAndRender()` — берёт `state.allPlaces`, применяет filter/sort, пишет в `state.places`, рендерит
   - При смене client-side параметров вызывать `applyAndRender()` (без нового API запроса — мгновенная реакция)
   - При смене server-side параметров (category) вызывать `fetchAndRender()` (новый API запрос)

4. **Inline search с geocode:** при вводе нового адреса в inline search, делать `api.geocode()` для получения новых `lat/lng`, затем `fetchAndRender()`. Не просто обновлять `state.q` — backend может не поддерживать текстовый поиск.

## Ключевые моменты
- Анимация обновления grid ≠ реальное изменение данных. Если карточки перерисовываются с `opacity:0→1` но данные те же — пользователь видит "моргание" без результата
- При удалении pill (сброс фильтра) тоже использовать `applyAndRender()`, не `fetchAndRender()`
- `state.allPlaces` — неизменный массив от API. `state.places` — результат фильтрации/сортировки. Это позволяет мгновенно переключать фильтры без повторных запросов
