# Разделение HTML монолита на отдельные файлы

## Когда применять
Когда проект содержит один большой `.html` файл (500+ строк) со встроенными `<style>` и `<script>` блоками, и его нужно разделить на отдельные файлы для удобства разработки.

## Шаги

1. **Прочитать весь файл** — файл может быть обрезан при чтении, использовать `offset` для постраничного чтения до конца. Не приступать к разделению пока не прочитан весь файл.

2. **Создать `styles.css`** — извлечь содержимое между `<style>` и `</style>` (без самих тегов). Заменить `<style>...</style>` в HTML на `<link rel="stylesheet" href="styles.css">`.

3. **Создать `app.js`** — извлечь содержимое между `<script>` и `</script>` (без самих тегов). Заменить `<script>...</script>` в HTML на `<script src="app.js"></script>`.

4. **Переписать `index.html`** — оставить только чистую HTML-разметку с подключением внешних файлов. Разместить `<link>` в `<head>`, `<script>` — в конце `<body>`.

5. **Проверить nginx/staic serving** — убедиться что веб-сервер монтирует папку с файлами и отдает CSS/JS по тем же относительным путям.

## Ключевые моменты
- Путь `styles.css` и `app.js` — относительные, без подпапок, чтобы nginx отдавал их из того же каталога
- В `<link>` не нужен `type="text/css"` — устаревший атрибут
- `<script src="app.js"></script>` ставится **после** всего HTML (в конце `<body>`), чтобы DOM был доступен
- Inline `onclick` обработчики в HTML продолжают работать — они вызывают глобальные функции из `app.js`
